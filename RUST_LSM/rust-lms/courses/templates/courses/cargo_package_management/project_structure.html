📁 Cargo Project Structure

📌 Key Concept: Cargo follows a standard project layout convention. This consistency makes it easy for Rust developers to understand and navigate any Rust project.

Standard Package Layout

A typical Rust package has this structure:

my_package/ ├── Cargo.toml # Package manifest ├── Cargo.lock # Dependency lock file ├── src/ # Source code │ ├── main.rs # Binary entry point │ ├── lib.rs # Library entry point │ └── bin/ # Additional binaries │ ├── tool1.rs │ └── tool2.rs ├── tests/ # Integration tests │ └── integration_test.rs ├── benches/ # Benchmarks │ └── my_benchmark.rs ├── examples/ # Example programs │ └── simple.rs ├── target/ # Build output (generated) │ ├── debug/ │ └── release/ ├── build.rs # Build script (optional) └── README.md # Documentation

Core Files

Cargo.toml

The manifest file containing package metadata and dependencies.

[package] name = "my_package" version = "0.1.0" edition = "2021" [dependencies] serde = "1.0"

Cargo.lock

Automatically generated file with exact dependency versions.

  • Binary projects: Commit to version control
  • Library projects: Add to .gitignore

README.md

Project documentation displayed on crates.io and GitHub.

Source Directory (src/)

src/main.rs

Entry point for a binary crate (executable).

fn main() { println!("Hello, world!"); }

src/lib.rs

Root module for a library crate.

pub fn add(a: i32, b: i32) -> i32 { a + b } #[cfg(test)] mod tests { use super::*; #[test] fn test_add() { assert_eq!(add(2, 2), 4); } }

Hybrid Packages

A package can have both lib.rs and main.rs:

my_package/ ├── Cargo.toml └── src/ ├── lib.rs # Library code └── main.rs # Binary using the library
// src/main.rs use my_package::add; // Import from lib.rs fn main() { println!("2 + 2 = {}", add(2, 2)); }

Multiple Binaries (src/bin/)

Create multiple executable programs in one package:

my_package/ ├── Cargo.toml └── src/ ├── lib.rs ├── main.rs # Default binary └── bin/ ├── client.rs # Additional binary └── server.rs # Additional binary

Run specific binaries:

$ cargo run # Runs main.rs $ cargo run --bin client # Runs src/bin/client.rs $ cargo run --bin server # Runs src/bin/server.rs

Tests Directory (tests/)

Integration tests go in the tests/ directory:

my_package/ ├── Cargo.toml ├── src/ │ └── lib.rs └── tests/ ├── integration_test.rs └── common/ └── mod.rs # Shared test utilities
// tests/integration_test.rs use my_package::add; #[test] fn test_integration() { assert_eq!(add(5, 5), 10); }

Run tests:

$ cargo test

Examples Directory (examples/)

Demonstrate how to use your library:

my_package/ └── examples/ ├── basic.rs └── advanced.rs
// examples/basic.rs use my_package::add; fn main() { let result = add(10, 20); println!("Result: {}", result); }

Run examples:

$ cargo run --example basic $ cargo run --example advanced

Benchmarks Directory (benches/)

Performance benchmarks (requires nightly Rust):

my_package/ └── benches/ └── my_benchmark.rs
// benches/my_benchmark.rs #![feature(test)] extern crate test; use test::Bencher; use my_package::add; #[bench] fn bench_add(b: &mut Bencher) { b.iter(|| add(2, 2)); }
$ cargo +nightly bench

Build Scripts (build.rs)

Optional script that runs before compilation:

// build.rs fn main() { println!("cargo:rerun-if-changed=build.rs"); println!("cargo:rustc-env=BUILD_TIME={}", "2024-01-01"); }

Use cases:

  • Compile C/C++ code
  • Generate code
  • Check system dependencies
  • Set environment variables

Target Directory (target/)

Cargo places all build artifacts here:

target/ ├── debug/ # Debug builds │ ├── my_package # Binary executable │ ├── deps/ # Compiled dependencies │ └── libmy_package.rlib # Library artifact └── release/ # Release builds (--release) ├── my_package └── deps/
💡 Note: Always add target/ to .gitignore. It's large and can be regenerated.

Module Organization

Organize code into modules using files and directories:

src/ ├── main.rs ├── lib.rs └── utils/ ├── mod.rs # Module root ├── parser.rs # Submodule └── formatter.rs # Submodule
// src/lib.rs pub mod utils; // src/utils/mod.rs pub mod parser; pub mod formatter; // src/utils/parser.rs pub fn parse(input: &str) -> Vec { // ... }

Or use the newer style (Rust 2018+):

src/ ├── main.rs ├── lib.rs ├── utils.rs # Module root └── utils/ ├── parser.rs └── formatter.rs

Complete Real-World Example

my_web_app/ ├── Cargo.toml ├── Cargo.lock ├── README.md ├── .gitignore ├── src/ │ ├── main.rs # Application entry point │ ├── lib.rs # Library code │ ├── config.rs # Configuration module │ ├── models/ │ │ ├── mod.rs │ │ ├── user.rs │ │ └── post.rs │ ├── handlers/ │ │ ├── mod.rs │ │ └── api.rs │ └── bin/ │ └── cli.rs # CLI tool ├── tests/ │ ├── integration_test.rs │ └── common/ │ └── mod.rs ├── examples/ │ ├── basic.rs │ └── advanced.rs └── benches/ └── performance.rs

💡 Key Takeaways

  • Cargo.toml - Package manifest
  • src/main.rs - Binary entry point
  • src/lib.rs - Library root module
  • src/bin/ - Additional binaries
  • tests/ - Integration tests
  • examples/ - Usage examples
  • benches/ - Performance benchmarks
  • target/ - Build output (don't commit)
  • Following conventions makes projects easy to understand
