🚀 Getting Started with Cargo

📌 Prerequisites: Cargo comes bundled with Rust when you install it using rustup. If you haven't installed Rust yet, visit rustup.rs and follow the installation instructions.

1. Verifying Cargo Installation

First, verify that Cargo is installed:

$ cargo --version
cargo 1.83.0 (5ffbef321 2024-10-29)

If you see a version number, Cargo is installed correctly!

2. Creating a New Project with cargo new

The easiest way to start a Rust project is with cargo new:

Creating a Binary (Executable) Project

$ cargo new hello_world $ cd hello_world
Created binary (application) `hello_world` package

Creating a Library Project

$ cargo new my_library --lib
Created library `my_library` package
💡 Binary vs Library:
  • Binary (default): Creates an executable application with src/main.rs
  • Library (--lib): Creates a reusable library with src/lib.rs

3. Understanding Project Structure

Let's explore what cargo new created:

hello_world/ ├── Cargo.toml # Package manifest ├── .gitignore # Git ignore file └── src/ └── main.rs # Main source file

Cargo.toml Contents

[package] name = "hello_world" version = "0.1.0" edition = "2021" [dependencies]
  • name: Package name (used when importing)
  • version: Current version (follows SemVer)
  • edition: Rust edition (2015, 2018, 2021, or 2024)
  • [dependencies]: External crates your project needs

src/main.rs Contents

fn main() { println!("Hello, world!"); }

4. Building Your Project

Compile your project with cargo build:

$ cargo build
Compiling hello_world v0.1.0 (/path/to/hello_world) Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.50s

This creates:

  • target/debug/hello_world - Your compiled executable
  • Cargo.lock - Dependency lock file

Debug vs Release Builds

# Debug build (default, faster compilation, slower execution) $ cargo build # Release build (slower compilation, optimized for speed) $ cargo build --release
When to use each:
  • Debug: Development, testing, fast iteration
  • Release: Production, benchmarking, final distribution

5. Running Your Project

Instead of building and then running separately, use cargo run:

$ cargo run
Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.01s Running `target/debug/hello_world` Hello, world!

cargo run automatically:

  1. Checks if code has changed
  2. Compiles if necessary
  3. Runs the resulting binary

Passing Arguments

# Arguments after -- are passed to your program $ cargo run -- arg1 arg2

6. Checking Your Code

Use cargo check to quickly verify code compiles without generating an executable:

$ cargo check
Checking hello_world v0.1.0 (/path/to/hello_world) Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.15s
✅ Pro Tip: cargo check is much faster than cargo build. Use it frequently during development to catch errors quickly!

7. Using cargo init

If you already have a directory, use cargo init instead of cargo new:

$ mkdir my_project $ cd my_project $ cargo init
Created binary (application) package

This creates Cargo.toml and src/ in the current directory.

Differences between cargo new and cargo init

  • cargo new <name>: Creates a new directory with the project
  • cargo init: Initializes project in existing directory

8. Additional Options

Specify Edition

$ cargo new my_project --edition 2021

Version Control Options

# Don't initialize git repository $ cargo new my_project --vcs none # Use git (default) $ cargo new my_project --vcs git # Use mercurial $ cargo new my_project --vcs hg

Custom Name

# Create project with name different from directory $ cargo new my-app --name my_app

9. First Complete Example

Let's create and run a complete project:

# Step 1: Create project $ cargo new greeting_app $ cd greeting_app # Step 2: Edit src/main.rs # (use your favorite editor) # Step 3: Build and run $ cargo run

Modified src/main.rs:

use std::io; fn main() { println!("What's your name?"); let mut name = String::new(); io::stdin() .read_line(&mut name) .expect("Failed to read line"); println!("Hello, {}!", name.trim()); }

Running the Program:

$ cargo run Compiling greeting_app v0.1.0 Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.75s Running `target/debug/greeting_app` What's your name? Alice Hello, Alice!

10. Understanding the Target Directory

After building, your project structure looks like:

hello_world/ ├── Cargo.toml ├── Cargo.lock ├── src/ │ └── main.rs └── target/ └── debug/ ├── hello_world # Your executable ├── deps/ # Compiled dependencies └── build/ # Build scripts output
📝 Note: The target/ directory can be safely deleted. Cargo will recreate it when you build again. It's typically excluded from version control.

11. Cleaning Build Artifacts

Remove compiled files to free up space:

$ cargo clean

This removes the entire target/ directory.

12. Getting Help

Cargo has built-in help for all commands:

# General help $ cargo help # Help for specific command $ cargo help build $ cargo help run $ cargo help new

Or use the --help flag:

$ cargo build --help $ cargo run --help

💡 Key Takeaways

  • cargo new <name> - Create a new project
  • cargo new <name> --lib - Create a library project
  • cargo init - Initialize project in current directory
  • cargo build - Compile your project
  • cargo run - Compile and run your project
  • cargo check - Quick syntax and type check
  • cargo build --release - Optimized production build
  • cargo clean - Remove build artifacts
  • Debug builds go in target/debug/
  • Release builds go in target/release/

⚠️ Common Beginner Mistakes

  • Running the binary directly instead of using cargo run
  • Forgetting to use --release for performance testing
  • Committing the target/ directory to version control
  • Not using cargo check for quick feedback during development
