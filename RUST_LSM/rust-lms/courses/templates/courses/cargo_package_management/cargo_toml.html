📄 Cargo.toml Guide

📌 What is Cargo.toml?
Cargo.toml is the manifest file written in TOML format that contains all metadata needed to compile your package.

Complete Example

[package] name = "my_awesome_project" version = "0.1.0" edition = "2021" authors = ["Your Name "] description = "A short description of my package" documentation = "https://docs.rs/my_awesome_project" homepage = "https://example.com/my_awesome_project" repository = "https://github.com/yourusername/my_awesome_project" readme = "README.md" license = "MIT OR Apache-2.0" keywords = ["web", "api", "async"] categories = ["web-programming"] [dependencies] serde = { version = "1.0", features = ["derive"] } tokio = { version = "1.0", features = ["full"] } reqwest = "0.11" [dev-dependencies] mockall = "0.11" [build-dependencies] cc = "1.0"

The [package] Section

Required Fields

Field Description Example
name Package name (used for imports) "my_project"
version Semantic version "0.1.0"
edition Rust edition "2021"

Optional Metadata

[package] authors = ["Alice ", "Bob "] description = "A library for doing awesome things" documentation = "https://docs.rs/my_crate" homepage = "https://my-crate.com" repository = "https://github.com/user/my_crate" readme = "README.md" license = "MIT" license-file = "LICENSE.txt" keywords = ["cli", "tools"] # Max 5 keywords categories = ["command-line-utilities"] publish = true # or false to prevent publishing

Dependencies

Simple Version

[dependencies] serde = "1.0" regex = "1.5"

Detailed Version

[dependencies] # Version with features serde = { version = "1.0", features = ["derive"] } # Git dependency my_lib = { git = "https://github.com/user/my_lib", branch = "main" } # Path dependency (local) my_local_crate = { path = "../my_local_crate" } # Optional dependency openssl = { version = "0.10", optional = true } # Platform-specific [target.'cfg(windows)'.dependencies] winapi = "0.3"

Version Requirements

Syntax Meaning Example
1.2.3 Exactly version 1.2.3 serde = "=1.2.3"
1.2 Compatible (^1.2.0) serde = "1.2"
^1.2.3 ≥1.2.3, <2.0.0 serde = "^1.2.3"
~1.2.3 ≥1.2.3, <1.3.0 serde = "~1.2.3"
* Any version serde = "*"

Dependency Types

[dev-dependencies]

Used only for tests, examples, and benchmarks:

[dev-dependencies] criterion = "0.5" proptest = "1.0" mockall = "0.11"

[build-dependencies]

Used only in build scripts (build.rs):

[build-dependencies] cc = "1.0" bindgen = "0.60"

Features

Conditional compilation flags:

[features] default = ["std"] std = [] serde_support = ["serde", "serde_json"] experimental = [] [dependencies] serde = { version = "1.0", optional = true } serde_json = { version = "1.0", optional = true }

Enable features:

$ cargo build --features serde_support $ cargo build --all-features $ cargo build --no-default-features

In code:

#[cfg(feature = "serde_support")] use serde::{Serialize, Deserialize};

Profiles

Customize compilation settings:

[profile.dev] opt-level = 0 # No optimizations [profile.release] opt-level = 3 # Maximum optimizations lto = true # Link-time optimization codegen-units = 1 [profile.test] opt-level = 1 # Some optimization for tests

Optimization Levels

  • 0 - No optimization (fast compile, slow runtime)
  • 1 - Basic optimization
  • 2 - Some optimization
  • 3 - Maximum optimization (slow compile, fast runtime)
  • "s" - Optimize for size
  • "z" - Optimize aggressively for size

Workspace Configuration

For projects with multiple crates:

[workspace] members = [ "crate1", "crate2", "tools/*", ] exclude = ["old-crate"] [workspace.dependencies] serde = "1.0" # Shared dependency version

Target-Specific Dependencies

Platform-conditional dependencies:

# Windows only [target.'cfg(windows)'.dependencies] winapi = "0.3" # Unix only [target.'cfg(unix)'.dependencies] nix = "0.26" # Specific architecture [target.'cfg(target_arch = "wasm32")'.dependencies] wasm-bindgen = "0.2"

Metadata Table

Custom metadata for tools:

[package.metadata.docs.rs] all-features = true rustdoc-args = ["--cfg", "docsrs"] [package.metadata.my_tool] setting1 = "value1" setting2 = 42

💡 Key Takeaways

  • Cargo.toml is written in TOML format
  • [package] section contains package metadata
  • [dependencies] for regular dependencies
  • [dev-dependencies] for test/development only
  • [build-dependencies] for build scripts
  • [features] for conditional compilation
  • [profile.*] for build configuration
  • Version requirements use SemVer conventions
