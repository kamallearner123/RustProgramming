Day 1 Morning: Introduction to Embedded Systems & Environment Setup

Introduction to Embedded Systems Programming

  • What are embedded systems?
    Embedded systems are specialized computing systems that perform dedicated functions within larger systems. Examples include automotive ECUs, IoT devices, medical equipment, and industrial controllers.
  • Challenges in embedded systems development
    Developers face resource constraints (CPU, memory, power), real-time requirements, hardware diversity, and debugging/testing difficulties.
  • Why Rust? Safety, reliability, and modern tooling
    Rust offers memory safety without garbage collection, zero-cost abstractions, concurrency without data races, and modern tooling (Cargo, rust-analyzer, Clippy).
    Contributors: Hanqi, Sergey
  • Rust community & resources for continued learning
    The Rust Embedded Working Group, Rust Embedded Book, forums, Discord, Matrix, and conferences provide ongoing support and learning opportunities.
    Contributors: Hanqi, Rodrigo

Setting up the Development Environment

  • Installing Rust for embedded systems development
    Use rustup to install Rust toolchains. Add target architectures (e.g., thumbv7em-none-eabihf) for cross-compilation.
  • Configuring toolchains & environment
    Install cross-compilation tools (cargo-binutils, llvm-tools-preview). Set up VS Code or your preferred IDE with rust-analyzer.
  • Exploring microcontroller families & platforms
    Learn about ARM Cortex-M, RISC-V, ESP32, AVR, and other platforms.
  • MCU ecosystem maturity & supported vendors
    ARM: STM32, NXP, Nordic, Microchip, etc. RISC-V: SiFive, Espressif, etc. Community-maintained HALs and BSPs.
    Contributors: Sergey, Rodrigo
  • References to real-world embedded Rust projects
    Explore Tock OS, RTIC, Ferrous Systems’ knurling, drone, embassy, and more.
    Contributors: Rodrigo, Sergey
