Exploring Microcontroller Families & Platforms

Rust embedded development supports a wide range of microcontroller architectures and platforms. Understanding the characteristics, strengths, and use cases of different MCU families helps you choose the right hardware for your project.

ARM Cortex-M Family

ARM Cortex-M processors are the most popular choice for embedded Rust development, offering excellent ecosystem support, comprehensive documentation, and mature tooling.

Cortex-M0/M0+/M1 - Entry Level

Architecture: ARMv6-M
Target: thumbv6m-none-eabi
CPU Frequency: Up to 48 MHz (typical)
Features:
  • Ultra-low power consumption
  • Minimal silicon area (cost-effective)
  • 32-bit instruction set (Thumb)
  • Simple interrupt handling
Use Cases: IoT sensors, wearables, battery-powered devices, simple control systems
Popular MCUs:
  • STM32F0: STMicroelectronics entry-level series
  • NRF51: Nordic Semiconductor Bluetooth LE
  • LPC11xx: NXP low-power series
  • SAMD21: Microchip (used in Arduino Zero)

Cortex-M3 - Mainstream

Architecture: ARMv7-M
Target: thumbv7m-none-eabi
CPU Frequency: Up to 120 MHz
Features:
  • Hardware divide instructions
  • Enhanced debugging (ETM trace)
  • Memory Protection Unit (MPU)
  • Better interrupt latency
Use Cases: Motor control, industrial automation, data acquisition, communication protocols
Popular MCUs:
  • STM32F1: Classic "Blue Pill" board
  • STM32L1: Low-power variant
  • LPC17xx: NXP mainstream series
  • EFM32: Silicon Labs energy-efficient MCUs

Cortex-M4/M4F - Digital Signal Processing

Architecture: ARMv7E-M
Target: thumbv7em-none-eabi (no FPU)
thumbv7em-none-eabihf (with FPU)
CPU Frequency: Up to 180 MHz
Features:
  • DSP instructions (SIMD)
  • Optional single-precision FPU
  • Saturating arithmetic
  • Fast multiply-accumulate
  • Memory Protection Unit
Use Cases: Audio processing, sensor fusion, motor control, machine learning inference, communication
Popular MCUs:
  • STM32F3: F3Discovery board (popular for learning)
  • STM32F4: High-performance with FPU
  • NRF52: Bluetooth 5.0 with strong Rust support
  • STM32L4: Ultra-low-power with FPU
  • TEENSY 4.x: i.MX RT1062 (600 MHz!)

Cortex-M7 - High Performance

Architecture: ARMv7E-M
Target: thumbv7em-none-eabihf
CPU Frequency: Up to 550 MHz
Features:
  • Superscalar dual-issue pipeline
  • 64-bit double-precision FPU
  • L1 instruction and data caches
  • Tightly Coupled Memory (TCM)
  • Advanced branch prediction
Use Cases: Real-time video processing, complex control systems, high-speed communication, edge AI
Popular MCUs:
  • STM32F7: High-performance series
  • STM32H7: Dual-core variants available
  • IMXRT10xx: NXP's crossover MCUs (up to 600 MHz)
  • SAME70: Microchip high-performance

Cortex-M33/M35P - Security & TrustZone

Architecture: ARMv8-M Mainline
Target: thumbv8m.main-none-eabi (no FPU)
thumbv8m.main-none-eabihf (with FPU)
CPU Frequency: Up to 160 MHz
Features:
  • TrustZone security technology
  • Secure/Non-secure memory regions
  • Hardware stack limit checking
  • Optional DSP and FPU
  • Co-processor interface
Use Cases: Secure IoT devices, payment systems, cryptography, secure boot, OTA updates
Popular MCUs:
  • STM32L5: Ultra-low-power with TrustZone
  • STM32U5: Latest ultra-low-power series
  • NRF5340: Dual-core Bluetooth with TrustZone
  • LPC55S6x: NXP secure MCUs

RISC-V Architecture

RISC-V is an open-source instruction set architecture gaining rapid adoption in embedded systems. Rust has excellent RISC-V support with growing ecosystem maturity.

RISC-V 32-bit Embedded

Targets: riscv32i-unknown-none-elf (base)
riscv32imac-unknown-none-elf (with multiply, atomic, compressed)
riscv32imc-unknown-none-elf (without atomic)
Features:
  • Open-source ISA (no licensing fees)
  • Modular instruction set extensions
  • Simple, clean architecture
  • Excellent code density
  • Growing ecosystem support
Popular MCUs:
  • GD32VF103: GigaDevice 108 MHz RISC-V
  • ESP32-C3: Espressif WiFi+BLE with Rust support
  • ESP32-C6: WiFi 6 + BLE 5
  • SiFive FE310: Freedom E310 (HiFive1 board)
  • CH32V: WCH low-cost RISC-V series
Rust Support:
  • riscv crate for core functionality
  • riscv-rt for runtime support
  • HAL crates for specific chips
  • embassy async runtime support

Popular Development Boards

Here are some popular development boards excellent for learning embedded Rust:

STM32F3DISCOVERY

MCU: STM32F303VCT6 (Cortex-M4F)

Features:

  • 72 MHz, 256KB Flash, 48KB RAM
  • Onboard sensors (gyro, accel, mag)
  • 8 LEDs, USB, ST-Link debugger

Why choose: Used in "The Embedded Rust Book", excellent documentation, affordable (~$15)

BBC micro:bit v2

MCU: NRF52833 (Cortex-M4F)

Features:

  • 64 MHz, 512KB Flash, 128KB RAM
  • Bluetooth 5.0, 5x5 LED matrix
  • Accelerometer, compass, microphone

Why choose: Educational focus, wireless capabilities, strong Rust support via microbit crate

Raspberry Pi Pico

MCU: RP2040 (Dual Cortex-M0+)

Features:

  • 133 MHz, 264KB RAM, 2MB Flash
  • Unique PIO (Programmable I/O)
  • USB device/host support

Why choose: Extremely affordable ($4), excellent rp-hal support, innovative PIO system

ESP32-C3

MCU: ESP32-C3 (RISC-V)

Features:

  • 160 MHz, 400KB RAM, 4MB Flash
  • WiFi 4 + Bluetooth 5.0 LE
  • 22 GPIOs, USB Serial/JTAG

Why choose: WiFi/BLE connectivity, RISC-V learning, growing esp-hal ecosystem

STM32F411 Black Pill

MCU: STM32F411CEU6 (Cortex-M4F)

Features:

  • 100 MHz, 512KB Flash, 128KB RAM
  • USB OTG, small form factor
  • Breadboard-friendly

Why choose: Very affordable (~$4), powerful, excellent stm32f4xx-hal support

Nordic NRF52840 DK

MCU: NRF52840 (Cortex-M4F)

Features:

  • 64 MHz, 1MB Flash, 256KB RAM
  • Bluetooth 5.0, NFC, USB
  • Onboard debugger

Why choose: Professional BLE development, excellent nrf-hal support, strong ecosystem

Vendor HAL Support

STMicroelectronics

  • stm32f0xx-hal, stm32f1xx-hal, stm32f3xx-hal, stm32f4xx-hal, stm32f7xx-hal
  • stm32l0xx-hal, stm32l1xx-hal, stm32l4xx-hal, stm32l5xx-hal
  • stm32h7xx-hal, stm32g0xx-hal, stm32g4xx-hal
  • Excellent documentation and community support

Nordic Semiconductor

  • nrf51-hal, nrf52840-hal, nrf9160-hal
  • Strong BLE and wireless protocol support
  • rubble - Pure Rust BLE stack

Raspberry Pi

  • rp-hal - Excellent RP2040 support
  • Unique PIO (Programmable I/O) abstractions
  • Very active community development

Espressif

  • esp-hal - ESP32-C3, ESP32-C6 support
  • WiFi and Bluetooth integration
  • esp-wifi - Rust WiFi driver
  • Rapidly maturing ecosystem

NXP

  • lpc8xx-hal, lpc55-hal
  • imxrt-hal for high-performance Cortex-M7
  • Growing ecosystem with commercial focus

Choosing the Right Platform

For Beginners:
  • STM32F3DISCOVERY - Best documentation, "The Embedded Rust Book" uses this
  • micro:bit v2 - Educational focus, wireless capabilities
  • Raspberry Pi Pico - Most affordable, unique features
For IoT/Wireless Projects:
  • ESP32-C3 - WiFi + BLE, RISC-V architecture
  • NRF52840 - Professional BLE development
  • STM32WL - LoRa and sub-GHz radio
For High Performance:
  • STM32H7 - Dual-core, up to 480 MHz
  • TEENSY 4.1 - 600 MHz Cortex-M7
  • i.MX RT1060 - Crossover MCU with high speed
For Open Hardware Enthusiasts:
  • ESP32-C3/C6 - Open documentation, WiFi/BLE
  • GD32VF103 - RISC-V alternative to STM32
  • SiFive HiFive1 - Open-source RISC-V reference

Next Steps

Once you've selected your microcontroller platform:

  • Install the appropriate Rust target for your architecture
  • Find the HAL crate for your specific MCU
  • Review vendor documentation and datasheets
  • Explore example projects in the HAL repository
  • Join the embedded Rust community on Matrix/Discord
