Installing Rust for Embedded Systems Development

Setting up Rust for embedded systems development requires installing the Rust toolchain along with specific targets and tools for cross-compilation. This guide walks you through the complete installation process.

Prerequisites

System Requirements

  • Operating System: Linux, Windows, or macOS
  • Internet connection for downloading tools
  • Sufficient disk space (approximately 2-3 GB)
  • Administrator/sudo access for system-level installations
Windows Users: You will need C++ Build Tools for Visual Studio 2019 or later. Download from the Microsoft website.

Step 1: Install Rustup

1Rustup is the official Rust toolchain installer and version manager. It allows you to install multiple Rust versions and manage toolchain components.

Installation Command

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
What this does: Downloads and runs the rustup installation script using secure HTTPS and TLS 1.2.

During Installation

The installer will present you with installation options:

1) Proceed with installation (default) 2) Customize installation 3) Cancel installation

For most users, selecting option 1 (default) is recommended. This installs:

  • The stable Rust compiler (rustc)
  • Cargo (Rust's package manager and build tool)
  • Standard library
  • Rustup for managing toolchains

Verify Installation

After installation, restart your terminal and verify the installation:

rustc --version cargo --version rustup --version

Expected output:

rustc 1.83.0 (90b35a623 2024-11-26) cargo 1.83.0 (5ffbef321 2024-10-29) rustup 1.27.1 (54dd3d00f 2024-04-24)
Note: Ensure you have Rust 1.31 or newer for embedded development. The version numbers shown above are examples and will vary.

Step 2: Install Embedded Targets

2Embedded systems use different processor architectures than desktop computers. You need to add cross-compilation support for your target microcontroller architecture.

ARM Cortex-M Architectures

ARM Cortex-M processors are the most common in embedded Rust development. Choose the appropriate target based on your microcontroller:

Cortex-M0/M0+/M1

Architecture: ARMv6-M

rustup target add thumbv6m-none-eabi

Common MCUs: STM32F0, NRF51, LPC11xx

Cortex-M3

Architecture: ARMv7-M

rustup target add thumbv7m-none-eabi

Common MCUs: STM32F1, STM32L1, LPC17xx

Cortex-M4/M7 (No FPU)

Architecture: ARMv7E-M

rustup target add thumbv7em-none-eabi

Common MCUs: STM32F3 (basic), STM32F4 (basic)

Cortex-M4F/M7F (With FPU)

Architecture: ARMv7E-M + HW Float

rustup target add thumbv7em-none-eabihf

Common MCUs: STM32F3, STM32F4, STM32F7, STM32H7

Cortex-M23

Architecture: ARMv8-M Baseline

rustup target add thumbv8m.base-none-eabi

Common MCUs: STM32L0, NRF9160

Cortex-M33/M35P (No FPU)

Architecture: ARMv8-M Mainline

rustup target add thumbv8m.main-none-eabi

Common MCUs: STM32L5, NRF5340

Cortex-M33F/M35PF (With FPU)

Architecture: ARMv8-M Mainline + HW Float

rustup target add thumbv8m.main-none-eabihf

Common MCUs: STM32L5, STM32U5, NRF5340

Cortex-M55

Architecture: ARMv8.1-M + MVE

rustup target add thumbv8m.main-none-eabihf

Note: Uses same target as M33F with additional vector extensions

RISC-V Targets

For RISC-V microcontrollers like GigaDevice GD32V and SiFive FE310:

rustup target add riscv32imac-unknown-none-elf rustup target add riscv32imc-unknown-none-elf

Verify Target Installation

List all installed targets:

rustup target list --installed

Step 3: Install Essential Tools

3Several additional tools are essential for embedded development:

cargo-binutils

Provides LLVM tools (objdump, nm, size, etc.) as Cargo subcommands:

cargo install cargo-binutils rustup component add llvm-tools

Why use cargo-binutils?

  • Single installation command works on all operating systems
  • Supports all architectures that rustc supports
  • No need for architecture-specific GNU binutils
  • Integrates seamlessly with Cargo workflow

cargo-generate

Creates new projects from templates:

cargo install cargo-generate
Linux Note: On Ubuntu/Debian, you may need to install dependencies first:
sudo apt install libssl-dev pkg-config

cargo-embed and probe-rs

Modern tools for flashing and debugging embedded devices:

cargo install cargo-embed probe-rs-cli

Features:

  • Flash firmware to microcontrollers
  • Real-time logging via RTT (Real-Time Transfer)
  • GDB server for debugging
  • Supports multiple debug probes (ST-Link, J-Link, CMSIS-DAP)

cargo-flash and cargo-size

Additional utilities for embedded development:

cargo install cargo-flash cargo install cargo-size

Step 4: Install Debugging Tools

4Debugging embedded systems requires additional software:

GDB with ARM Support

Linux (Ubuntu/Debian):

sudo apt install gdb-multiarch

macOS:

brew install armmbed/formulae/arm-none-eabi-gcc

Windows:

Download ARM GCC from the ARM Developer website.

OpenOCD (Optional)

Open On-Chip Debugger for advanced debugging scenarios:

Linux:

sudo apt install openocd

macOS:

brew install openocd

QEMU for ARM

Emulator for testing without hardware:

Linux:

sudo apt install qemu-system-arm

macOS:

brew install qemu

Step 5: Verify Complete Setup

5Verify all components are correctly installed:

# Check Rust toolchain rustc --version cargo --version # Check installed targets rustup target list --installed # Check cargo tools cargo --list | grep -E "binutils|generate|embed|flash|size" # Check LLVM tools cargo objdump --version # Check GDB arm-none-eabi-gdb --version # or gdb-multiarch --version # Check QEMU (optional) qemu-system-arm --version

Common Installation Issues

Issue: cargo install fails with linker errors

Solution: Install build dependencies:
  • Ubuntu/Debian: sudo apt install build-essential
  • macOS: Install Xcode Command Line Tools: xcode-select --install
  • Windows: Install Visual Studio Build Tools

Issue: rustup command not found after installation

Solution: The installation adds Rust to your PATH, but you need to restart your terminal or run:
source $HOME/.cargo/env

Issue: Permission denied errors on Linux/macOS

Solution: Don't use sudo with cargo install. If you have permission issues, fix cargo directory ownership:
sudo chown -R $USER:$USER ~/.cargo

Next Steps

With your Rust embedded development environment installed, you're ready to:

  • Configure your toolchain for specific targets
  • Create your first embedded Rust project
  • Set up debugging with your hardware probe
  • Explore microcontroller platforms and ecosystems
Pro Tip: Keep your toolchain updated regularly:
rustup update
This ensures you have the latest features, bug fixes, and security patches.
