🔢 Data Types in Rust

📌 Key Concept: Rust is a statically typed language, meaning all variable types must be known at compile time. The compiler can usually infer types, but explicit annotations are sometimes needed.

Type System Overview

Rust has two main categories of data types:

  • Scalar Types: Represent a single value
  • Compound Types: Group multiple values

1. Scalar Types

Scalar types represent a single value. Rust has four primary scalar types:

Integer Types

Integers are numbers without fractional components.

Length Signed Unsigned Range (Signed)
8-bit i8 u8 -128 to 127
16-bit i16 u16 -32,768 to 32,767
32-bit i32 u32 -2,147,483,648 to 2,147,483,647
64-bit i64 u64 -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807
128-bit i128 u128 Very large range
arch isize usize Depends on architecture (32 or 64-bit)
fn main() { let x: i32 = 42; // 32-bit signed integer (default) let y: u8 = 255; // 8-bit unsigned integer let z: isize = 1000; // Architecture-dependent // Number literals with type suffix let a = 98_222; // Underscore for readability let b = 0xff; // Hexadecimal let c = 0o77; // Octal let d = 0b1111_0000; // Binary let e = b'A'; // Byte (u8 only) println!("x = {}, y = {}, z = {}", x, y, z); }
⚠️ Integer Overflow: In debug mode, Rust checks for integer overflow and panics. In release mode, it wraps around (modular arithmetic). Use methods like wrapping_add(), checked_add(), overflowing_add(), or saturating_add() for explicit behavior.

Floating-Point Types

Floating-point numbers have decimal points.

Type Precision Use Case
f32 Single precision (32-bit) When speed matters more than precision
f64 Double precision (64-bit) Default, better precision
fn main() { let x = 2.0; // f64 (default) let y: f32 = 3.0; // f32 // Floating-point operations let sum = x + 5.0; let difference = 95.5 - 4.3; let product = 4.0 * 30.0; let quotient = 56.7 / 32.2; println!("sum: {}, difference: {}", sum, difference); }
💡 Note: All floating-point types are signed. Rust follows IEEE-754 standard.

Boolean Type

The Boolean type has two possible values: true and false.

fn main() { let t = true; let f: bool = false; // With explicit type annotation // Booleans are often used in conditionals if t { println!("It's true!"); } // Boolean operations let is_greater = 5 > 3; let is_equal = 5 == 5; let combined = t && is_greater; println!("is_greater: {}, is_equal: {}", is_greater, is_equal); }

Booleans are one byte in size.

Character Type

Rust's char type represents a Unicode Scalar Value.

fn main() { let c = 'z'; let z: char = 'ℤ'; let heart_eyed_cat = '😻'; // Chars are 4 bytes and can represent any Unicode character println!("Character: {}", heart_eyed_cat); // Note: Single quotes for char, double quotes for strings let char_type = 'A'; // char let string_type = "A"; // &str (string slice) }
💡 Important:
  • char is 4 bytes in size
  • Represents Unicode Scalar Values (U+0000 to U+D7FF and U+E000 to U+10FFFF)
  • More than just ASCII!

2. Compound Types

Compound types can group multiple values into one type.

Tuple Type

A tuple groups together multiple values with different types. Tuples have a fixed length.

fn main() { // Creating a tuple let tup: (i32, f64, u8) = (500, 6.4, 1); // Destructuring a tuple let (x, y, z) = tup; println!("The value of y is: {}", y); // Accessing tuple elements by index let five_hundred = tup.0; let six_point_four = tup.1; let one = tup.2; println!("First element: {}", five_hundred); // Empty tuple (unit type) let unit: () = (); }
💡 Unit Type: The empty tuple () is called the unit type and represents an empty value or empty return type. Functions that don't return a value implicitly return ().

Array Type

Arrays in Rust have a fixed length and all elements must be the same type.

fn main() { // Array with explicit type let a: [i32; 5] = [1, 2, 3, 4, 5]; // Array initialized with same value let b = [3; 5]; // [3, 3, 3, 3, 3] // Accessing array elements let first = a[0]; let second = a[1]; println!("First: {}, Second: {}", first, second); // Array length println!("Array length: {}", a.len()); // Iterating over array for element in &a { println!("Value: {}", element); } }
⚠️ Index Out of Bounds: Rust checks array bounds at runtime. Accessing an invalid index will cause a panic.
let a = [1, 2, 3]; let element = a[10]; // ❌ Panic: index out of bounds

Arrays vs Vectors

Feature Array Vector (Vec<T>)
Size Fixed at compile time Can grow/shrink at runtime
Memory Stack-allocated Heap-allocated
Syntax [T; N] Vec<T>
Use Case Fixed size, fast access Dynamic size needed

3. Type Inference

Rust can often infer types based on usage:

fn main() { let x = 5; // Inferred as i32 (default integer type) let y = 2.0; // Inferred as f64 (default float type) let z = true; // Inferred as bool let c = 'a'; // Inferred as char // Sometimes inference needs help let parsed: i32 = "42".parse().expect("Not a number!"); // Or use turbofish syntax let parsed = "42".parse::().expect("Not a number!"); }

4. Type Aliases

Create custom names for existing types:

type Kilometers = i32; type Point = (f64, f64); fn main() { let distance: Kilometers = 100; let origin: Point = (0.0, 0.0); println!("Distance: {} km", distance); }

5. Practical Examples

Example 1: Working with Different Types

fn main() { // User profile let username: &str = "rustacean"; let age: u8 = 25; let height: f32 = 5.9; let is_verified: bool = true; let initial: char = 'R'; println!("User: {}, Age: {}, Height: {}ft", username, age, height); println!("Verified: {}, Initial: {}", is_verified, initial); }

Example 2: RGB Color Tuple

fn main() { let red: (u8, u8, u8) = (255, 0, 0); let green = (0, 255, 0); let blue = (0, 0, 255); println!("Red: RGB({}, {}, {})", red.0, red.1, red.2); // Destructuring let (r, g, b) = red; println!("Red components: R={}, G={}, B={}", r, g, b); }

Example 3: Temperature Readings Array

fn main() { let temperatures: [f64; 7] = [72.5, 73.1, 71.8, 74.2, 72.9, 73.5, 71.6]; // Calculate average let mut sum = 0.0; for temp in &temperatures { sum += temp; } let average = sum / temperatures.len() as f64; println!("Average temperature: {:.2}°F", average); }

💡 Key Takeaways

  • Scalar types: integers, floats, booleans, characters
  • Compound types: tuples and arrays
  • Rust is statically typed but has excellent type inference
  • Arrays are fixed-size, use Vec for dynamic collections
  • Type annotations use : syntax (e.g., let x: i32 = 5;)
  • Default integer type is i32, default float is f64
