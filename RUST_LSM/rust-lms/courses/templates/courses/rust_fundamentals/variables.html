📦 Variables & Mutability

📌 Key Concept: In Rust, variables are immutable by default. This is a fundamental design choice that promotes safety and prevents accidental data modification.

1. Declaring Variables

Variables in Rust are declared using the let keyword:

fn main() { let x = 5; println!("The value of x is: {}", x); }

This creates an immutable variable named x with the value 5.

2. Immutability by Default

Once a value is bound to a variable, you cannot change that value:

fn main() { let x = 5; x = 6; // ❌ ERROR: cannot assign twice to immutable variable }
⚠️ Compiler Error: The Rust compiler will prevent you from compiling this code, catching bugs at compile time rather than runtime!

Why Immutability?

  • Safety: Prevents accidental data modification
  • Concurrency: Makes it easier to reason about concurrent code
  • Performance: Compiler can make optimizations knowing data won't change
  • Predictability: Code behavior is more predictable

3. Mutable Variables

When you need to modify a variable, explicitly declare it as mutable using mut:

fn main() { let mut x = 5; println!("The value of x is: {}", x); x = 6; // ✅ This works! println!("The value of x is now: {}", x); }
✅ Output: 
The value of x is: 5
The value of x is now: 6

The mut keyword explicitly signals that this variable's value may change, making your intentions clear to other developers.

4. Shadowing

Rust allows you to declare a new variable with the same name as a previous variable. The new variable shadows the previous one:

fn main() { let x = 5; let x = x + 1; // Shadows the first x { let x = x * 2; // Shadows within this scope println!("The value of x in the inner scope is: {}", x); // 12 } println!("The value of x is: {}", x); // 6 }
✅ Output: 
The value of x in the inner scope is: 12
The value of x is: 6

Shadowing vs Mutability

Feature Shadowing (let) Mutability (mut)
Creates new variable ✅ Yes ❌ No
Can change type ✅ Yes ❌ No
Requires let ✅ Yes ❌ No
Scoped ✅ Yes ❌ No (same variable)

Example: Changing Types with Shadowing

fn main() { let spaces = " "; // String slice let spaces = spaces.len(); // Number - Different type! println!("Number of spaces: {}", spaces); // 3 }

This is not possible with mutability:

fn main() { let mut spaces = " "; spaces = spaces.len(); // ❌ ERROR: expected `&str`, found `usize` }

5. Constants

Constants are values that are bound to a name and cannot change. Unlike variables:

  • Must be annotated with a type
  • Can be declared in any scope, including global
  • Can only be set to a constant expression (not the result of runtime computation)
  • Use const keyword
  • Named using SCREAMING_SNAKE_CASE by convention
const THREE_HOURS_IN_SECONDS: u32 = 60 * 60 * 3; const MAX_POINTS: u32 = 100_000; fn main() { println!("Max points: {}", MAX_POINTS); }

Constants vs Immutable Variables

Feature Constants (const) Immutable Variables (let)
Type annotation ✅ Required ❌ Optional (inferred)
Can use mut ❌ Never ✅ Yes (let mut)
Runtime values ❌ No ✅ Yes
Global scope ✅ Yes ❌ Limited
Shadowing ❌ No ✅ Yes

6. Variable Naming Conventions

// Variables and functions: snake_case let user_name = "Alice"; let total_count = 42; // Constants: SCREAMING_SNAKE_CASE const MAX_SIZE: usize = 1024; const PI: f64 = 3.14159; // Types, traits, enums: PascalCase (covered later) // struct UserProfile { ... }

7. Practical Examples

Example 1: Counter with Mutability

fn main() { let mut counter = 0; for i in 1..=5 { counter += i; println!("Counter: {}", counter); } println!("Final count: {}", counter); // 15 }

Example 2: Shadowing for Transformations

fn main() { let input = " hello "; println!("Original: '{}'", input); let input = input.trim(); println!("Trimmed: '{}'", input); let input = input.to_uppercase(); println!("Uppercase: '{}'", input); // 'HELLO' }

Example 3: Configuration Constants

const BUFFER_SIZE: usize = 8192; const TIMEOUT_SECONDS: u64 = 30; const DEBUG_MODE: bool = true; fn main() { if DEBUG_MODE { println!("Debug: Buffer size is {} bytes", BUFFER_SIZE); println!("Debug: Timeout is {} seconds", TIMEOUT_SECONDS); } }

💡 Key Takeaways

  • Variables are immutable by default in Rust
  • Use mut to explicitly allow modification
  • Shadowing creates a new variable, allowing type changes
  • Constants are always immutable and must be compile-time values
  • Immutability promotes safety and helps prevent bugs
