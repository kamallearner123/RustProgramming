⚙️ Functions in Rust

📌 Key Concept: Functions are the primary way to organize and reuse code in Rust. Rust uses the fn keyword to declare functions. The main function is the entry point of every Rust program.

1. Function Basics

Functions are declared using the fn keyword:

fn main() { println!("Hello, world!"); another_function(); } fn another_function() { println!("Another function."); }
✅ Output: 
Hello, world!
Another function.

Function Naming Convention

  • Use snake_case for function names
  • Examples: calculate_total, fetch_user_data, is_valid

2. Function Parameters

Functions can have parameters (also called arguments). You must declare the type of each parameter.

fn main() { print_labeled_measurement(5, 'h'); } fn print_labeled_measurement(value: i32, unit_label: char) { println!("The measurement is: {}{}", value, unit_label); }
✅ Output: The measurement is: 5h

Multiple Parameters

fn calculate_area(width: f64, height: f64) { let area = width * height; println!("The area is: {} square units", area); } fn main() { calculate_area(10.5, 20.3); }

3. Return Values

Functions can return values. The return type is declared after an arrow ->.

fn five() -> i32 { 5 // No semicolon = this is an expression that returns the value } fn main() { let x = five(); println!("The value of x is: {}", x); }

Explicit Return Statement

fn plus_one(x: i32) -> i32 { return x + 1; // Explicit return with 'return' keyword } fn main() { let result = plus_one(5); println!("Result: {}", result); // 6 }
⚠️ Important: Most Rust functions return values using expressions (no semicolon). Adding a semicolon turns an expression into a statement, which doesn't return a value.
fn plus_one(x: i32) -> i32 { x + 1; // ❌ ERROR: semicolon makes this return () instead of i32 }

4. Expressions vs Statements

Statements

Statements are instructions that perform an action and do not return a value.

fn main() { let y = 6; // Statement: doesn't return a value }

Expressions

Expressions evaluate to a resulting value.

fn main() { let y = { let x = 3; x + 1 // Expression: returns 4 (no semicolon!) }; println!("The value of y is: {}", y); // 4 }
Feature Statement Expression
Returns value ❌ No ✅ Yes
Ends with semicolon ✅ Yes ❌ No (usually)
Examples let x = 5; 5 + 3, { x + 1 }

5. Multiple Return Values

Use tuples to return multiple values:

fn calculate_dimensions(width: f64, height: f64) -> (f64, f64, f64) { let area = width * height; let perimeter = 2.0 * (width + height); let diagonal = (width.powi(2) + height.powi(2)).sqrt(); (area, perimeter, diagonal) } fn main() { let (area, perimeter, diagonal) = calculate_dimensions(10.0, 20.0); println!("Area: {}", area); println!("Perimeter: {}", perimeter); println!("Diagonal: {:.2}", diagonal); }

6. Function Signatures

A complete function signature specifies:

// Function signature breakdown: // fn - function keyword // add - function name // (a: i32, b: i32) - parameters with types // -> i32 - return type fn add(a: i32, b: i32) -> i32 { a + b }

Unit Return Type

Functions that don't return a value implicitly return () (unit type):

fn print_message(msg: &str) { println!("{}", msg); // Implicitly returns () } // Equivalent to: fn print_message_explicit(msg: &str) -> () { println!("{}", msg); }

7. Early Returns

Use return to exit a function early:

fn divide(numerator: f64, denominator: f64) -> f64 { if denominator == 0.0 { println!("Error: Division by zero!"); return 0.0; // Early return } numerator / denominator } fn main() { let result = divide(10.0, 0.0); println!("Result: {}", result); }

8. Function Examples

Example 1: Temperature Converter

fn fahrenheit_to_celsius(f: f64) -> f64 { (f - 32.0) * 5.0 / 9.0 } fn celsius_to_fahrenheit(c: f64) -> f64 { (c * 9.0 / 5.0) + 32.0 } fn main() { let temp_f = 98.6; let temp_c = fahrenheit_to_celsius(temp_f); println!("{}°F is {:.1}°C", temp_f, temp_c); let back_to_f = celsius_to_fahrenheit(temp_c); println!("{:.1}°C is {:.1}°F", temp_c, back_to_f); }

Example 2: Mathematical Functions

fn square(n: i32) -> i32 { n * n } fn is_even(n: i32) -> bool { n % 2 == 0 } fn factorial(n: u32) -> u32 { if n <= 1 { return 1; } n * factorial(n - 1) // Recursive call } fn main() { println!("Square of 5: {}", square(5)); println!("Is 4 even? {}", is_even(4)); println!("Factorial of 5: {}", factorial(5)); }

Example 3: String Formatting

fn format_full_name(first: &str, last: &str) -> String { format!("{} {}", first, last) } fn greet_user(name: &str, age: u32) -> String { format!("Hello, {}! You are {} years old.", name, age) } fn main() { let full_name = format_full_name("Alice", "Smith"); println!("{}", full_name); let greeting = greet_user("Bob", 30); println!("{}", greeting); }

Example 4: Data Validation

fn is_valid_age(age: i32) -> bool { age >= 0 && age <= 150 } fn is_valid_email(email: &str) -> bool { email.contains('@') && email.contains('.') } fn validate_user_input(age: i32, email: &str) -> bool { is_valid_age(age) && is_valid_email(email) } fn main() { let age = 25; let email = "user@example.com"; if validate_user_input(age, email) { println!("User input is valid!"); } else { println!("Invalid user input!"); } }

9. Function Pointers

Functions can be stored in variables and passed as arguments:

fn add(a: i32, b: i32) -> i32 { a + b } fn multiply(a: i32, b: i32) -> i32 { a * b } fn apply_operation(a: i32, b: i32, op: fn(i32, i32) -> i32) -> i32 { op(a, b) } fn main() { let result1 = apply_operation(5, 3, add); let result2 = apply_operation(5, 3, multiply); println!("5 + 3 = {}", result1); println!("5 * 3 = {}", result2); }

10. Diverging Functions

Functions that never return use the ! type:

fn panic_function() -> ! { panic!("This function never returns!"); } fn infinite_loop() -> ! { loop { println!("This runs forever..."); } }
💡 Note: The ! type (called "never") indicates that a function never returns normally. This is useful for functions that always panic or loop forever.

💡 Key Takeaways

  • Functions are declared with fn keyword
  • Use snake_case for function names
  • Parameters must have explicit types
  • Return type is specified with -> Type
  • The last expression (without semicolon) is the return value
  • Expressions return values, statements don't
  • Use tuples to return multiple values
  • Functions can be passed as arguments (function pointers)
