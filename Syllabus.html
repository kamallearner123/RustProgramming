Rust Programming Course Syllabus

Designed for fresh college graduates with C/C++ knowledge. The course is 15 days long, with 8 hours of sessions per day.

Day Topics Examples Assignments
1 Introduction to Rust, Setup, and Basics "Hello, World!" program, using `cargo`. Write a program to print the Fibonacci series.
2 Ownership, Borrowing, and Data Types Examples of ownership transfer and borrowing. Reverse a string using ownership rules.
3 Functions and Control Flow Calculator program, `if` and `match` examples. Write a program to check if a number is prime.
4 Structs, Enums, and Pattern Matching Student record system using structs and enums. Create a to-do list manager with structs and enums.
5 Collections: Vectors and Strings Word frequency counter using a vector. Sort an array of integers provided by the user.
6 Error Handling File reader with `Result` and `Option` usage. Parse and validate email addresses.
7 Traits and Generics Generic sorting and custom traits for shapes. Find the maximum in a generic list.
8 Modules and Crates Creating a modularized program, using `rand` crate. Create a dice-rolling simulator.
9 Iterators and Closures Filtering even numbers, closures for squares. Write a custom iterator for a range of numbers.
10 Concurrency Thread creation, message passing with `mpsc`. Producer-consumer problem using threads.
11 Smart Pointers Examples with `Box`, `Rc`, and `RefCell`. Create a linked list using `Box` and `Rc`.
12 Async Programming Using async/await with `tokio` to fetch APIs. Download files concurrently using async tasks.
13 Unsafe Rust Raw pointers, interfacing with C libraries. Wrap a C library using Rust FFI.
14 Mini Project Build a CLI-based todo list manager. Enhance the todo list manager with concurrency.
15 Final Project and Wrap-up Build a multithreaded web server in Rust. Document the project and present the implementation.

Mini Projects

  • CLI-based Todo List Manager: Manage tasks with file storage, and allow concurrent access.
  • Web Server: Develop a basic multithreaded web server to handle HTTP requests.
