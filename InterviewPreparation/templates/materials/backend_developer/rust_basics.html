Example: Error Handling with Result and Option

Demonstrates safe error handling using Result for division and Option for searching a value.


fn divide(a: i32, b: i32) -> Result<i32, String> {
    if b == 0 {
        Err(String::from("Division by zero"))
    } else {
        Ok(a / b)
    }
}

fn find_value(arr: &[i32], target: i32) -> Option<usize> {
    arr.iter().position(|&x| x == target)
}

fn main() {
    // Result example
    match divide(10, 2) {
        Ok(result) => println!("Result: {}", result),
        Err(e) => println!("Error: {}", e),
    }

    match divide(10, 0) {
        Ok(result) => println!("Result: {}", result),
        Err(e) => println!("Error: {}", e),
    }

    // Option example
    let numbers = vec![1, 2, 3, 4, 5];
    match find_value(&numbers, 3) {
        Some(index) => println!("Found at index: {}", index),
        None => println!("Value not found"),
    }
}

Why This Example?

Shows Result for error handling and Option for optional values, using match for control flow. Discuss how these replace null pointers for safety.

Interview Tip

Explain match vs. if let and the ? operator for error propagation.

2. Data Structures

Concepts Covered

  • Vec<T>: Dynamic arrays.
  • HashMap<K, V>: Key-value store.
  • enum: Custom types with variants.
  • struct: Custom data structures.

Example: Task Manager

A task manager using Vec, HashMap, enum, and struct.


use std::collections::HashMap;

#[derive(Debug)]
enum TaskStatus {
    Todo,
    InProgress,
    Done,
}

#[derive(Debug)]
struct Task {
    id: u32,
    description: String,
    status: TaskStatus,
}

fn main() {
    // Vector to store tasks
    let mut tasks: Vec<Task> = Vec::new();
    tasks.push(Task {
        id: 1,
        description: String::from("Write Rust code"),
        status: TaskStatus::Todo,
    });
    tasks.push(Task {
        id: 2,
        description: String::from("Test application"),
        status: TaskStatus::InProgress,
    });

    // HashMap to map task ID to description
    let mut task_map: HashMap<u32, String> = HashMap::new();
    for task in &tasks {
        task_map.insert(task.id, task.description.clone());
    }

    // Print tasks
    println!("Tasks: {:?}", tasks);
    println!("Task Map: {:?}", task_map);

    // Find task by ID
    if let Some(desc) = task_map.get(&1) {
        println!("Task ID 1: {}", desc);
    }
}

Why This Example?

Shows practical use of Vec, HashMap, enum, and struct in a task manager. Discuss ownership and cloning in the interview.

Interview Tip

Explain when to use Vec vs. HashMap and how enums enable algebraic data types.

3. Modules and Crates

Concepts Covered

  • Modules for code organization, pub for visibility.
  • Cargo for dependencies and testing.

Example: Modular Code with Tests

Organizes math operations in a module with tests.


// src/lib.rs
pub mod math {
    pub fn add(a: i32, b: i32) -> i32 {
        a + b
    }

    pub fn subtract(a: i32, b: i32) -> i32 {
        a - b
    }
}

#[cfg(test)]
mod tests {
    use super::math::{add, subtract};

    #[test]
    fn test_add() {
        assert_eq!(add(2, 3), 5);
    }

    #[test]
    fn test_subtract() {
        assert_eq!(subtract(5, 3), 2);
    }
}

// src/main.rs
use my_crate::math;

fn main() {
    println!("Add: {}", math::add(10, 5));
    println!("Subtract: {}", math::subtract(10, 5));
}

Why This Example?

Demonstrates module organization, visibility, and testing. Assumes a crate named my_crate.

Interview Tip

Discuss pub vs. private, Cargo dependency management, and integration tests.

4. Generics and Traits

Concepts Covered

  • Generics with <T>.
  • Custom traits and trait bounds.

Example: Generic Stack with Trait

A generic stack with a custom Printable trait.


use std::fmt::Debug;

trait Printable {
    fn print(&self);
}

#[derive(Debug)]
struct Stack<T> {
    items: Vec<T>,
}

impl<T: Debug> Printable for Stack<T> {
    fn print(&self) {
        println!("Stack: {:?}", self.items);
    }
}

impl<T> Stack<T> {
    fn new() -> Self {
        Stack { items: Vec::new() }
    }

    fn push(&mut self, item: T) {
        self.items.push(item);
    }

    fn pop(&mut self) -> Option<T> {
        self.items.pop()
    }
}

fn main() {
    let mut int_stack: Stack<i32> = Stack::new();
    int_stack.push(1);
    int_stack.push(2);
    int_stack.print();

    let mut str_stack: Stack<String> = Stack::new();
    str_stack.push(String::from("hello"));
    str_stack.print();
}

Why This Example?

Shows generics, custom traits, and trait bounds in a reusable stack implementation.

Interview Tip

Discuss monomorphization and impl Trait vs. dyn Trait.

5. Async Rust

Concepts Covered

  • Futures, async/await.
  • Tokio runtime for concurrency.

Example: Async HTTP Client

Fetches multiple URLs concurrently using Tokio and reqwest.


use tokio;
use reqwest;

#[tokio::main]
async fn main() -> Result<(), reqwest::Error> {
    let urls = vec![
        "https://api.github.com",
        "https://jsonplaceholder.typicode.com/posts/1",
    ];

    let mut handles = vec![];

    // Spawn async tasks for each URL
    for url in urls {
        let handle = tokio::spawn(async move {
            let response = reqwest::get(url).await?;
            let body = response.text().await?;
            Ok::<_, reqwest::Error>(body)
        });
        handles.push(handle);
    }

    // Wait for all tasks to complete
    for handle in handles {
        match handle.await.unwrap() {
            Ok(body) => println!("Response: {}", &body[..50]),
            Err(e) => println!("Error: {}", e),
        }
    }

    Ok(())
}

Why This Example?

Shows concurrent URL fetching with Tokio. Add tokio = { version = "1", features = ["full"] } and reqwest = "0.11" to Cargo.toml.

Interview Tip

Explain std::thread vs. tokio::spawn and async I/O benefits.

6. Performance

Concepts Covered

Example: Macro and Iterator Optimization

Uses a logging macro and iterators for efficient data processing.


macro_rules! log {
    ($($arg:tt)*) => {
        println!("[INFO] {}", format!($($arg)*));
    };
}

fn process_numbers(numbers: Vec<i32>) -> i32 {
    numbers
        .into_iter()
        .filter(|&x| x > 0)
        .map(|x| x * 2)
        .sum()
}

fn main() {
    let numbers = vec![-1, 2, 3, -4, 5];
    log!("Processing numbers: {:?}", numbers);
    
    let result = process_numbers(numbers);
    log!("Result: {}", result);
}

Why This Example?

Shows a custom macro and iterator chain for performance.

Interview Tip

Discuss iterator laziness and macro expansion at compile time.

Example: Error Propagation with Documentation

Reads a file with proper error handling and documentation.


use std::fs::File;
use std::io::{self, Read};

/// Reads a file's contents into a string.
///
/// # Errors
/// Returns an `io::Error` if the file cannot be read.
fn read_file(path: &str) -> Result<String, io::Error> {
    let mut file = File::open(path)?;
    let mut contents = String::new();
    file.read_to_string(&mut contents)?;
    Ok(contents)
}

fn main() {
    match read_file("example.txt") {
        Ok(contents) => println!("File contents: {}", contents),
        Err(e) => eprintln!("Error reading file: {}", e),
    }
}

Why This Example?

Shows the ? operator, doc comments, and idiomatic error handling.

Interview Tip

Explain why unwrap is dangerous and how Clippy enforces best practices.
